Overview

What the internet actually enables

The internet is the infrastructure that allows connected systems to exchange data. The web is one of the services built on top of that infrastructure, enabling browsers and servers to communicate and deliver websites, applications, and digital experiences.

01

Networks connect devices

Information can travel between computers because networks create routes for communication.

02

Servers deliver resources

Servers host web pages, files, APIs, and services that browsers request.

03

Browsers render content

Browsers interpret code and transform it into visual interfaces users can interact with.

04

Protocols define communication

Rules like HTTP and HTTPS structure how messages are sent and received on the web.

Web flow

What happens when you open a website

A website visit is a quick exchange between client and server. The browser sends a request, the server returns resources, and the browser assembles the final result on screen.

Client

Browser sends request

You enter a URL or click a link, and the browser requests data from a server.

Server

Server processes request

The server locates the requested resource and prepares the response.

Render

Browser builds interface

HTML, CSS, and JavaScript are parsed so the browser can display the final page.

Core stack

The three main frontend technologies

These technologies work together, but each one solves a different part of the problem.

HTML

Defines the structure and meaning of content using elements like headings, sections, paragraphs, forms, links, and media.

CSS

Controls the visual layer of the interface, including spacing, layout, colors, responsiveness, animations, and presentation rules.

JavaScript

Handles logic and interactivity, making interfaces dynamic and able to react to user input.

Architecture

Frontend and backend play different roles

The frontend focuses on what users see and interact with. The backend handles the logic, data processing, storage, and server-side operations behind the scenes.

Frontend

The part of the application that runs in the browser and shapes the user experience.

  • Layout and interface structure
  • Visual styling and responsiveness
  • User interactions and client-side behavior

Backend

The part of the application that runs on the server and manages data and logic.

  • Authentication and authorization
  • Business rules and processing
  • Database access and API responses

Glossary

Quick terms worth understanding early

A small set of definitions that helps reduce confusion when reading documentation or tutorials.

Browser

Software that requests, interprets, and displays web pages.

Server

A machine or service that stores resources and returns them when requested.

URL

The address used to locate a resource on the web.

HTTP

The protocol that defines how browsers and servers communicate.

API

An interface that allows systems or applications to exchange data.

Responsive Design

An approach that adapts interfaces to different screen sizes and devices.

Frequently asked questions

Common beginner questions

These are some common doubts people usually have when starting to learn the web.

Should I learn HTML before CSS and JavaScript?

Yes. HTML is usually the best starting point because it provides the structure of a webpage.

What is the difference between the internet and the web?

The internet is the infrastructure, while the web is one of the services built on top of it.

Do I need backend knowledge to begin with frontend?

No. You can start with frontend first, but backend knowledge becomes valuable as you build more complete applications.
